انواع Vulnerability :
privilege escalation : هجوم بسبب اخطاء في كتابة كود نظام التشغيل
Zero-Day Attack : ثغرة مطور البرنامج او النظام او الخ لا يعرف عنها احيانا تكون متواجدة بالانظمة القديمة المتوقف الدعم عنها
Default settings : الاعدادات التلقائية تعتبر اعدادات ضعيفة من ناحية الامان 
Unsecure root accounts : عدم وجود باسوورد للروت او باسوورده ضعيف والروت دائما يعتبر المدير او السوبر يوزر الذي يمتلك اعلى الصلاحيات 
Open permissons : الصلاحيات المفتوحة المسموحة للكل 
Open  ports and services : المنافذ والخدمات المفتوحة 
خطوات لتقليل الاستغلال للمنافذ
- تحديد اشخاص عن طريق عناوينهم للسماح لهم بعمل وصول 
- اغلاق المنافذ التي لا يتم استعمالها
Unsecure protocols : استخدام بروتوكولات غير امنة مثل http 
Weak encryption : استخدام انواع التشفير الضعيفة السهل فكها او كسرها 
Errors : الاخطاء التي تظهر عندما تعمل خطا ما مثل رسالة تنبيه بان الباسوورد خطا مم يعطي معلومة للمهاجم بان اليوزر صحيح لكن كلمة السر خاطئة
بعض المصطلحات : 
Data Breach : تغيير الصلاحيات للمعلومات كالسماح للكل بقراءتها او التعديل عليها او حذفها الخ
Data Exfilteration : تسريب للبيانات او تعتبر الطرق او الادوات التي تم من خلالها نقل للبيانات من جهاز الضحية لجهاز المهاجم 
Network Vulnerability Scanning Tools :
Nessus ( Paid and free ) + OpenVAS ( Free ) : تقوم بفحص اجهزة الشبكة كاملة فهي تحمل لستة بالثغرات المشهورة 
كيفية عمل Vulnaribility scanner :
يقوم بعمل detection ويبدا بالتعرف على الاجهزة
ثم يقوم بمسح الاجهزة بذاتها من ناحية الامان
ثم يقوم بعمل تقرير لك كامل عن ما وجد
تقدم لك طرق علاج الثغرات
ما هو الفرق بين Website و Web application? 
Website : موقع ثابت لا يتغير كل ما قمت بفتحه
Web Application : موقع يتغير ويتحدث باستمرار
Web Application Scanner Tools : 
Nikto : تعمل مسح على اشهر هجمات الويب 
Common Vulnerability and Exposure ( CVE ) : معجم للثغرات توضع برقم معين مثل
CVE-2023-1234 
: Scanانواع الـ
Non-Credential scan : محاكاة لفحص شخص من خارج الشبكة لا يملك باسوورد او يوزر للهدف 
Credential scan : محاكاة لفحص شخص من داخل الشبكة 
مختبر الاختراق لديه حدود لعمله او ما تسمى ( Scope ) وقواعد
اشكال اختبار الاختراق : 
Black Box : اختبار اختراق بدون اعطاء مختبر الاختراق اية معلومات عن اي شيء كانه مخترق خارجي ( شائع اكثر )
White Box : اعطاء معلومات كاملة لمختبر الاختراق وجعله يعمل اختبار اختراق كانه مخترق داخلي ( شائع اكثر )
Gray Box : اختبار اختراق مع معلومات قليلة وبسيطة كانه شخص موجود بالشبكة لكن لا يملك صلاحيات
